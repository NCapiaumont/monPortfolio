BugZilla

Présentation

Bugzilla est un logiciel libre de système de suivi de problèmes avec interface web, développé et utilisé par l'organisation Mozilla. Il permet le suivi de bogues ou de « demande d'amélioration » sous la forme de « tickets ».

Logiciel de type serveur, architecture trois tiers, il est écrit en langage Perl. Il est disponible sous UNIX (Linux, BSD, etc.) et est distribué sous tri-licence MPL/LGPL/GPL.

Il est utilisé par de nombreuses organisations pour suivre le développement de nombreux logiciels, sur Internet ou dans des réseaux privés. Les plus connus sont la Mozilla Foundation, Facebook, la Wikimedia Foundation, la NASA, YAHOO, GNOME, KDE, Red Hat, Novell et Mandriva.

InterEurope

Ralentissement BDD InterEurope

Nous nous trouvons dans le context de InterEurope. Où la base qui est en production va commencer a ralentir, nous allons voir les causes possibles de ralentissement ainsi qu'une solution pour palier à ce problème.

Causes

La cause du ralentissement est la table vol 65 000 vols par jour, qui passeront a 2 000 000 de vols par jour. Ce qui fait énormément de requêtes et donc qui va dans le futur ralentir la base de donnée quand il y aura beaucoup de vols.

Beaucoup trop de vols inutiles dans la table vol il serait

Solution

Création d'un autre serveur de base de données avec les mêmes tables, qui serait une copie en temps réél de l'autre serveur de base de données ce qui permettrai de découper/séparer les requetes, pour que le serveur soit moins surchargé. Un serveur recevrai exclusivement les requetes selects et l'autre serveur recevrait les inserts, delete et update.

Création d'une table archiveVol pour archiver des vols anciens pour ne pas surcharger la table vol. Création d'une table paramètre avec un champ archivage. Grace a un crontab mise a jour de la date dans la table paramètre, avec un trigger archivage des anciens vols de la table vol dans la table archivageVol.
